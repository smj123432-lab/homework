# 마크다운 문법

### 1. 마크다운이란?

마크다운은 일반 텍스트 문서에 서식 요소를 추가하는데 사용할 수 있는 가벼운 마크업 언어이다.
존 그루버가 2004년에 개발하였으며 현재 전세계에서 가장 널리 사용되는 마크업 언어 중 하나이다.

마크다운은 최근 깃 허브 덕분에 각광 받기 시작했는데, 깃 허브의 저장소에 관한 정보를 기록하는 README.md와 같은 깃 허브를 사용하는 사람이면 누구나 가장 먼저 접하게 되는 문서이기 때문이다.

장단점 파트에서 다시 설명하겠지만 마크다운을 통해 설치방법, 소스코드 설멸 그리고 이슈 등을 간단하게 기록하고 가독성을 높일 수 있다는 강점이 부각되며 점점 여러 곳으로 퍼져가게 되었다.

Microsoft Word와 같은 애플리케이션에서는 버튼을 클릭하여 단어와 구문을 서식 지정하고 변경 사항을 즉시 확인할 수 있지만, 마크다운은 그렇지 않다. 마크다운 서식 파일을 만들 때는 텍스트에 마크다운 구문을 추가하여 어떤 단어와 구문을 다르게 표시해야 하는지 지정한다.

### 2. 어떻게 작동하는가?

![마크다운의구성과정](https://mdg.imgix.net/assets/images/markdown-flowchart.png?auto=format&fit=clip&q=40&w=1080)
<font size = 1  >출처: https://www.markdownguide.org/getting-started/ </font>

1. 파일을 만든다. 파일 확장자는 '.' .md또는 '.'여야 한다 .markdown.
2. Markdown 애플리케이션에서 Markdown 파일을 연다.
3. Markdown 애플리케이션을 사용하여 Markdown 파일을 HTML 문서로 변환한다.
4. 웹 브라우저에서 HTML 파일을 보거나 Markdown 애플리케이션을 사용하여 PDF와 같은 다른 파일 형식으로 변환한다.

하지만 사용자 관점에서 사용하는 애플리케이션에 따라 프로세스가 다소 다를 수 있다고 한다.

예를 들어, Dillinger는 1~3단계를 하나의 매끄러운 인터페이스로 통합했다. 왼쪽 창에 입력하기만 하면 렌더링된 출력이 오른쪽 창에 마법처럼 나타난다. 하지만 정적 웹사이트 생성기가 있는 텍스트 편집기와 같은 다른 도구를 사용하면 프로세스가 훨씬 더 명확하게 보인다.

### 3. 장단점 및 사용처

#### 장점

- 간경하다
- 별도의 도구없이 작성 가능하다.
- 다양한 형태로 변환이 가능하다.
- 텍스트로 저장되기 때문에 용량이 적다
- 텍스트 파일이기에 버전관리시스템을 이용하여 변경 이력을 관리할 수 있다.
- 지원하는 프로그램과 플랫폼이 다양하다

#### 단점

- 표준이 없다.
- 표준이 없기에 도구에 따라 변환 방식이나 생성물이 다르다
- 모든 html 마크업을 대신하지는 못한다.

#### 사용하는 곳

메모장이나 전용 에디터까지 많은 곳에서 활용할 수 있다.

문법이 쉽기때문에 에디터를 굳이 사용할 필요는 없다.
마크다운 문법은 지원하는 모든 곳에서 사용할 수 있으며, 일반 블로그나 워드프레스, [벨로그](https://velog.io/) 외에 [슬랙](https://slack.com/)이나 [노션](https://www.notion.so/)과 같은 서비스에서 사용할 수 있다.

### 4. 마크다운 문법

#### 제목

제목을 나타내는 문법은
=======(제목), -------(부제목) : 문서의 대제목과 소제목
(# ~ ######): 글머리를 지정하는 문법 6개의 글머리까지 지정할 수 있다.

html 태그
h1 ~ h6 태그

#### 강조

_ 텍스트_: 이탈릭체(기울임)
(\*\* \*\*): 글의 두꺼움
(기울임과 두꺼움은 혼용해서 사용이 가능하다.)
~~ 텍스트~~: 취소 선
(밑줄은 마크다운에서 지원하지 않는다고 하니 직접 html 태그를 이용해야 한다.) -> `<u></u>`

html 태그
이탈릭 -> `<em></em>`
두껍게 -> `<strong></strong>`
취소선 -> `<del></del>`

#### 목록

**순서가 있는 리스트**
1,2,3,4 를 적는다.
ex)

1.  마
2.  크
3.  다
4.  운

**순서가 없는 리스트**
\*,-,+
ex)

- 사

* 용

- 법

html 태그
`순서 o -> <ol></ol>`
`순서 x -> <ul></ul>`

#### 링크

**사용법**

사용문법 1 :
`[Title](link)`
적용 예: [Google](https://google.com)

기본적으로 이 사용하나 js에서 변수를 선언하는 것처럼 참조 링크를 사용할 수도 있다.

사용문법 2 :
`[태그 1][참조 링크1]`
`[참조 링크1]: www.google.com`

html 태그
`<a href></a>`

#### 이미지

링크와 비슷하나 앞에 ! 를 붙인다.

`![대체 텍스트](이미지 주소)`

![대체 텍스트][참조]
[참조]: 이미지 주소

##### 이미지에 링크 추가

`[![텍스트](이미지 주소)](링크)`

와 같이 입력하면 사진을 눌렀을때 입력한 링크로 들어갈 수 있다.

#### 인라인

백틱 ( \` )을 사용하여 인라인 코드로 강조하여 표현한다.
`<background-color: black;>`

#### 블록

백틱( \` )을 세 번 이상 입력하여 코드를 명시한다.
코드 블록의 종료 백틱 개수는 시작 백틱 개수와 같아야한다.

```
멋쟁이 사자처럼 프론트엔드 부트캠프 16기
 과제
마크다운 내용 정리 및 회고 작성
```

#### 표

테이블 헤더를 구분하기 위해, 3개 이상의 -(hyphen/dash) 기호를 사용한다.

테이블 헤더를 구분하며 :(Colons) 기호를 추가해 셀 (열/칸) 안에 내용을 정렬할 수 있다.

| 헤더 | 헤더 | 헤더 |
| ---- | ---- | ---- |
| 셀   | 셀   | 셀   |
| 셀   | 셀   | 셀   |

#### 인용블럭

```
> 첫 번째
>> 두 번째
>>> 세 번째
```

> 첫 번째
>
> > 두 번째
> >
> > > 세 번쨰
> > >
> > > > 네 번째

인용블럭도 물론 다른 요소들과 함꼐 사용할 수 있다.
